学习笔记

笔记

  1. web搭建模式：

    • 子域名模式：网站一样，子域名不一样，是不同目标
      • www.xiaodi8.com zblog asp程序
      • xiaodi.xiaodi8.com wordpress php程序
    • 端口模式：子域名以及网站名一样，端口不一样，是不同目标
      • www.xiaodi8.com zblog asp程序
      • www.xiaodi8.com:8080 wordpress php程序
    • 目录模式：不同目录是不同的目标

  2. 开源-源码不可见，加密，语言特性

  3. 路由访问：

    • 常规：访问该目录下的指定文件 URL和文件目录对应上
    • URL和文件目录对应不上，要根据配置路由决定

  4. 站库分离：

    • 原理：源码数据库不在同服务器工
    • 存储：其他服务器上数据库&云数据库产品
    • 影响：数据被单独存放，能连接才可影响数据

  5. 数据库（MySQL/Redis）

    • MySQL：利用 union select 注入获取数据、破解数据库密码哈希、提权（如UDF提权）；
    • Redis：未授权访问后写入公钥/计划任务提权、数据泄露；
    • 防护要点：开启数据库防火墙、使用预编译语句防注入、限制数据库账户权限。

  6. Web服务器（Nginx/Apache）

    • 安全配置要点：关闭目录浏览、隐藏版本号、配置反向代理防直接访问后端；
    • 常见漏洞：Nginx解析漏洞（如 *.php.xxx 解析为PHP）、Apache多后缀解析漏洞、缓存区溢出漏洞；
    • 渗透测试方法：利用 nmap 扫描版本，结合EXP测试对应漏洞；检查 robots.txt 、 .htaccess 等配置文件泄露。

  7. 应用服务器（Tomcat/Node.js）

    • Tomcat安全：默认弱口令（tomcat/tomcat）、管理后台未授权访问、WAR包部署漏洞；
    • Node.js安全： eval() 函数注入、依赖包漏洞（如 npm audit 检测）、跨站脚本漏洞；
    • 测试要领：尝试访问管理后台、上传恶意WAR包/脚本，利用工具检测依赖漏洞。

  8. 前后端分离

    • 原理:前端JS框架，API传输数据
    • 影响:
      • 1、前端页面大部分不存在漏洞
      • 2、后端管理大部分不在同域名
      • 3、获得权限有可能不影响后端

  9. 正反向代理：

    • 正向代理：为客户端服务，客户端主动建立代理访问目标(不代理不可达)
    • 反向代理：为服务端服务，服务端主动转发数据给可访问地址(不主动不可达)
    • 正向代理和反向代理都是解决访问不可达的问题，但由于反向代理中多出一个可以重定向解析的功能操作，导致反代理出的站点指向和真实应用毫无关系
    • 将真实的网站指向任意一个地方
    • 影响：访问目标只是一个代理，非真实应用服务器

  10. WAF

    • 原理：Web应用防火墙，防护保护Web
    • 影响：安全测试手法会受到拦截

  11. CDN

    • 原理：内容分发服务，提高访问速度
    • 影响：隐藏真实IP，导致信息收集目标错误

  12. OSS

    • 原理：云存储服务，提高资源文件加载速度
    • 影响：
    • 资源单独存储，修复上传漏洞，文件解析不复存在，同时也会有Accesskey泄漏安全问题

  13. 负载均衡：

    • 原理：分摊到多个操作单元上进行执行，共同完成工作任务
    • 影响：有多个服务器加载服务，测试过程中存在多个目标情况

  14. Docker核心组件的安全风险

    • 镜像安全：恶意镜像包含后门、基础镜像存在漏洞（如Alpine/Linux的系统漏洞）；
    • 容器安全：容器逃逸（利用内核漏洞突破容器隔离）、容器间网络互通导致横向渗透；
    • 宿主机安全：Docker守护进程（Docker Daemon）权限过高、挂载宿主机目录导致文件篡改。

  15. Docker渗透测试步骤

    1. 信息收集：扫描目标端口（2375/tcp为Docker远程API端口），测试未授权访问；
    2. 镜像检测：拉取目标镜像，用 docker scan 检测漏洞，分析镜像内的配置文件/敏感数据；
    3. 容器测试：进入容器后尝试提权、逃逸（如利用 cgroup / cap_sys_admin 权限）；
    4. 宿主机渗透：通过挂载的宿主机目录写入恶意文件，或利用Docker API创建特权容器。

  16. Docker安全防护要点

    • 禁用Docker远程API的未授权访问，配置TLS认证；
    • 使用非root用户运行容器，限制容器的内核能力（ –cap-drop ）；
    • 定期更新镜像与Docker引擎，清理无用容器/镜像。

  17. 红蓝对抗视角的实战要领

    1. 前后端分离架构中，优先通过接口模糊测试（如参数类型篡改、边界值测试）发现漏洞；
    2. Docker环境渗透需掌握“容器→宿主机”的逃逸思路，重点关注内核漏洞与权限配置；
    3. 红蓝对抗中，可利用前后端分离的接口逻辑漏洞制造横向渗透，或通过Docker容器接管目标服务器。
